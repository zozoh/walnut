# 过滤器简介

`@join_many` 将其它的数据对象（可多个）加入本对象

# 用法

```bash
o @join_one
  [/path/${to}]     # 【选】一个路径模板，上下文是当前对象元数据
                    # 表示从此路径搜索数据
[-query {..}]       # 查询的更多条件
                    # 这个是一个模板，可以根据上下文obj
                    # 做 WnExplain，并且它一定会有 pid
                    # 优先从路径里获得，默认采用当前上下文对象的pid
  [-to myKey]       # 【必】存放查询结果的键
  [-sort {..}]      # 排序方式
  [-limit 10]       # 最大数量
  [-skip 0]         # 跳过多少记录
  #--------------------------------------
  [-mapping /path]  # [路径模板]将结果做转换，指向一个BeanMapping映射文件
  [-by "key"]       # 映射文件的某个键下面的内容才是映射信息
  [-only]           # 当映射时不在映射集合内的对象，将会被无视
  [-explain {..}]   # 【选2】将结果（mapping之后）再次转换成新值
  [-explainBy /path] # 【选2】[路径模板]比explain更优先，用一个JSON 文件
                     # 提供更加复杂的转换行为
```
